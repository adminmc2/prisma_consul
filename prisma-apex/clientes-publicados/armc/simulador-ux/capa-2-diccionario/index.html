Capa 2 — Diccionario Operativo y Payloads

Status: En desarrollo por Executor 3

Esta capa define el contrato de datos del flujo vigente: respuesta automática en WhatsApp según elección del usuario; si no queda satisfecho, escalado a atención humana con contexto completo; y convergencia final al Súper Formulario.

1. Lead Captured — Ingreso inicial

Lead ingresa por WhatsApp con consulta inicial sobre un procedimiento o precio.

{
  "event_type": "LEAD_CAPTURED",
  "canal_origen": "WHATSAPP",
  "nombre": "Ana Maria",
  "telefono": "+525512345678",
  "interes_principal": "PRECIO",
  "comentario_inicial": "Me interesa una valoración para rinomodelación"
}

2. Auto Response Sent — Respuesta automática

Sistema responde automáticamente según bucket de interés (PRECIO, PROCEDIMIENTO, etc.). Template predefinido enviado por WhatsApp.

{
  "event_type": "AUTO_RESPONSE_SENT",
  "decision_bucket": "PRECIO",
  "template_id": "armc_precio_base_v1",
  "canal": "WHATSAPP",
  "satisfaccion": "PENDING"
}

3. Human Support Requested — Escalado humano

Si lead no queda satisfecho con respuesta automática, se escala a agente humano con contexto completo.

{
  "event_type": "HUMAN_SUPPORT_REQUESTED",
  "motivo": "NO_SATISFECHO_AUTO_RESPONSE",
  "contexto_transferido": {
    "resumen_conversacion": "Solicita más detalle sobre riesgos y recuperación",
    "template_previa": "armc_precio_base_v1"
  },
  "operacion_actual": "ATENCION_HUMANA"
}

4. Super Form Completed — Conversión a usuario

Lead completa súper formulario con datos preclinicos. Transición Lead → Usuario + cita agendada.

{
  "event_type": "SUPER_FORM_COMPLETED",
  "mutacion_estado": "LEAD_TO_USER",
  "preclinical_intake": {
    "alergias": "Penicilina",
    "antecedentes": "Sin cirugías previas",
    "riesgos_reportados": "Sensibilidad a anestésicos"
  },
  "appointment": {
    "status": "AGENDADA",
    "fecha_sugerida": "2026-05-20T10:00:00-06:00"
  }
}
